# Stable Diffusion v1.5 水彩画风格 LoRA 微调项目规划

## 1. 项目目标

本项目采用 LoRA 对 Stable Diffusion v1.5 进行水彩画风格微调，使模型在保持基本 prompt 响应能力的同时，生成更接近目标水彩画数据集的图像。

核心目标：

- 构建 100 张左右的水彩画风格训练集。
- 使用 LoRA 对 Stable Diffusion v1.5 进行小数据集风格微调。
- 比较基础模型与 LoRA 模型在风格一致性、prompt 一致性、多样性和过拟合风险上的差异。
- 形成可写进报告的定量指标、可视化样例和分析结论。

## 2. 总体技术路线

1. 收集并清洗水彩画风格图像。
2. 统一图像尺寸、命名和 caption。
3. 使用 Stable Diffusion v1.5 作为基础模型。
4. 使用 LoRA 进行风格微调。
5. 保存多个 checkpoint，例如 500 step、1000 step、1500 step、2000 step。
6. 用少量固定 prompts 生成对比图。
7. 计算核心指标并做人工观察。
8. 在报告中展示训练样例、生成对比、指标表格和过拟合分析。

## 3. 数据集规划

### 3.1 数据集主题

主题为水彩画风格。对于风格 LoRA，数据集不需要覆盖过多内容类别，重点是风格一致、画面质量稳定。

建议数据构成：

- 公共数据集 90 张，分为 5 种水彩风景子风格，每类 18 张。
- 每位成员在自己负责的子风格上额外补充 10 张。
- 每位成员最终训练集为 `公共 90 张 + 对应风格额外 10 张 = 100 张`。

5 种公共子风格规划如下：

| 类别 | 公共数量 | 个人补充 | 作用 |
| --- | ---: | ---: | --- |
| A 类：山湖 | 18 | A 额外 10 张 | 学习山地湖泊水彩风景 |
| B 类：雪景 | 18 | B 额外 10 张 | 学习雪地、冬季和冷色水彩风景 |
| C 类：小镇 | 18 | C 额外 10 张 | 学习水彩小镇和建筑风景 |
| D 类：日落 | 18 | D 额外 10 张 | 重点学习 sunset 氛围和夕照色彩 |
| E 类：花草 | 18 | E 额外 10 张 | 学习花草、河岸和自然前景 |

这样既能让 Group90 覆盖五种水彩风景方向，又能让每个人的 10 张数据在对应子风格上形成可分析的影响。

### 3.2 数据收集建议

优先选择：

- 清晰度较高、没有明显水印和大段文字的水彩图。
- 风格一致的图像，例如颜色晕染、纸张纹理、笔触边缘、低饱和或透明叠色特征明显的作品。
- 构图和主体有一定差异的图像，避免 100 张都属于同一种角度、同一种对象。
- 授权较清楚或可公开引用来源的图像，报告里可以说明数据来源。

不建议选择：

- 摄影图、厚涂插画、油画、二次元赛璐璐风格图像混入数据集。
- 带大面积边框、文字、签名、水印的图像。
- 分辨率过低、压缩痕迹明显或主体严重裁切的图像。
- 和自己负责的水彩子风格差异很大的个人 10 张图像。

建议在收集时同步记录每张图的文件名、来源、授权、主体类别和所属子风格。这样后面写报告和排查问题会方便很多。

### 3.3 数据处理

处理要求：

- 统一缩放或裁剪到 512 x 512。
- 尽量保留主体完整，不要为了裁剪强行切掉主要对象。
- 删除质量很差、风格明显不一致的图。
- 保留少量内容多样性，但不要为了多样性牺牲风格一致性。

caption 建议：

- 每张图使用统一风格触发词，例如 `watercolor style`。
- caption 中再加入简短主体描述，例如 `watercolor style, a house beside a river`。
- 不需要写得过长，核心是让模型知道“风格词”和“主体内容”的对应关系。

## 4. 个人 10 张数据的处理策略

课程要求中“90 张公共图像 + 每人 10 张图像”的含义，更适合理解为：公共 90 张先均衡覆盖 5 种子风格，每个人再在自己负责的子风格上额外补充 10 张，形成个人完整 100 张训练集。

推荐方案：

- 每位成员都使用 `公共 90 张 + 自己负责子风格的 10 张图像` 训练一个个人 LoRA。
- 如果时间或算力有限，小组可以先训练一个统一主模型，再让每个人基于自己的 100 张数据补做少量对比或分析。
- 不建议只用个人 10 张图像单独做主实验，因为 10 张太少，容易过拟合，生成结果也不稳定。

把个人 10 张混入公共 90 张通常不会有严重负面影响，因为它们只占 10%。但前提是这 10 张要与自己负责的公共子风格一致。如果个人 10 张风格差异很大，会让 LoRA 学到更混杂的视觉特征。

可选的对比实验：

| 训练集 | 作用 | 是否推荐 |
| --- | --- | --- |
| 公共 90 张图像，即 5 类各 18 张 | 小组共享基线 Group90 | 推荐 |
| 公共 90 张图像 + 对应风格额外 10 张图像 | 每个人的主实验 A100 至 E100 | 推荐 |
| 个人 10 张图像 | 观察小样本过拟合 | 只作为补充实验 |

如果想让个人 10 张更有存在感，可以在报告中单独分析“对应风格增强”效果，但不建议在训练时给它们过高权重，否则可能破坏整体水彩风景风格。

## 5. Prompt 设计

100 张图像的数据集不需要设计太多 prompt。prompt 太多会让评估工作膨胀，也会测试到训练集没有覆盖的内容。

建议使用 6 至 8 个固定 prompts，每个 prompt 使用 3 个固定 seeds。这样每个模型生成 18 至 24 张图，足够做指标和可视化。

prompt 设计原则：

- 覆盖训练集中主要出现的内容，不要故意测试太偏的主题。
- 包含 2 至 3 个训练集中常见主体，例如风景、建筑、人物。
- 包含 1 至 2 个稍微泛化的主体，用于观察模型是否保留基础生成能力。
- 基础模型和 LoRA 模型使用相同主体描述；LoRA 模型可以额外加入风格触发词。

示例 prompts：

- `watercolor style, a quiet village beside a river`
- `watercolor style, a snowy forest path`
- `watercolor style, a flower shop on a rainy street`
- `watercolor style, a portrait of a young woman`
- `watercolor style, a cat sitting near a window`
- `watercolor style, an old stone bridge in spring`

生成时固定：

- prompt 列表
- seed 或 seed 组
- 分辨率
- 采样器
- 采样步数
- CFG scale
- negative prompt

## 6. 训练方案

### 6.1 推荐训练设置

建议先采用保守设置，避免 100 张小数据集过拟合。

| 项目 | 推荐值 |
| --- | --- |
| 基础模型 | Stable Diffusion v1.5 |
| 微调方法 | LoRA |
| 分辨率 | 512 x 512 |
| LoRA rank | 8 或 16 |
| LoRA alpha | 与 rank 相同或略小 |
| UNet learning rate | 1e-4 左右 |
| Text Encoder | 初期不训练，或使用很小学习率 |
| batch size | 根据显存设为 1 至 4 |
| mixed precision | fp16 或 bf16 |
| checkpoint 间隔 | 每 500 step 保存一次 |
| 总训练步数 | 1000 至 2000 step 起步 |

不建议一开始直接训练到很高步数。对 100 张风格图来说，训练过久容易出现构图重复、风格过强、prompt 控制能力下降等问题。

### 6.2 checkpoint 选择

建议至少保存并比较：

- 500 step：观察是否开始出现水彩风格。
- 1000 step：通常是比较重要的候选点。
- 1500 step：观察风格是否更稳定。
- 2000 step：检查是否开始过拟合。

如果 2000 step 已经出现明显重复或过拟合，就不需要继续训练到 3000 step。最终模型不一定选择训练步数最高的 checkpoint，而应选择风格一致性、prompt 一致性和多样性最平衡的版本。

### 6.3 推荐实验组合

主实验：

| 模型 | 用途 |
| --- | --- |
| Stable Diffusion v1.5 Base | 基础对照 |
| LoRA 500 step | 早期微调效果 |
| LoRA 1000 step | 中期候选模型 |
| LoRA 1500 step | 强风格候选模型 |
| LoRA 2000 step | 过拟合风险检查 |

补充实验：

- 如果算力充足，可以对比 `90 张公共图像` 和 `90 + 10 张个人图像` 的结果。
- 如果报告需要展示小样本风险，可以训练一个 `仅 10 张个人图像` 的 LoRA，但只作为过拟合示例，不作为最终主模型。

## 7. 训练资源规划

### 7.1 硬件配置

本项目训练资源先定为单张 NVIDIA GeForce RTX 4090。

RTX 4090 具有 24GB 显存，足够完成 Stable Diffusion v1.5 的 512 x 512 LoRA 训练，也方便同时保存多个 checkpoint 和生成评估图。

建议训练配置：

| 项目 | 推荐值 |
| --- | --- |
| GPU | 单张 RTX 4090 24GB |
| 分辨率 | 512 x 512 |
| batch size | 4 起步，可根据显存调整到 6 或 8 |
| mixed precision | fp16 或 bf16 |
| xformers / memory efficient attention | 开启 |
| gradient checkpointing | 可不开启；如显存紧张再开启 |

### 7.2 训练时间估算

在单张 RTX 4090 上，100 张 512 x 512 图像的 LoRA 训练时间预计较短。具体速度取决于训练脚本、batch size、是否训练 text encoder、数据读取速度和保存 checkpoint 的频率。

粗略估算：

| 训练步数 | 预计训练时间 |
| --- | --- |
| 500 step | 5 至 10 分钟 |
| 1000 step | 10 至 20 分钟 |
| 1500 step | 15 至 30 分钟 |
| 2000 step | 20 至 40 分钟 |

如果同时训练 text encoder、开启更复杂的数据增强或频繁生成预览图，时间可能增加到上述估算的 1.5 至 2 倍。即使考虑环境准备和中间测试，单个 LoRA 主实验通常可以在 1 小时内完成。

如果每位成员都训练一个 `90 张公共图像 + 10 张个人图像` 的 LoRA，按每人 1000 至 2000 step 估计，每个成员大约需要 20 至 60 分钟，包括训练、生成样例和简单检查。

### 7.3 存储估算

大致存储需求：

- Stable Diffusion v1.5 基础模型：约数 GB。
- LoRA checkpoint：通常几十 MB 到数百 MB。
- 数据集：100 张 512 x 512 图像通常不到 1 GB。
- 生成结果与评估缓存：建议预留 5 GB 以上。
- 整体磁盘空间：建议预留 30 GB 至 50 GB。

### 7.4 生成与评估工作量

建议每个模型使用 6 个 prompts、每个 prompt 3 个 seeds：

- 每个模型生成 18 张图。
- Base + 4 个 LoRA checkpoint 共 5 个模型。
- 总生成图数量约 90 张。

这个规模足够支撑报告分析，也不会让人工评分和最近邻展示变得过重。

## 8. 精简版评估指标

报告中主打 4 个指标即可，不建议把 FID / KID 作为核心。原因是本项目数据集只有 100 张，FID / KID 对样本量比较敏感，容易得到不稳定或解释价值不高的结果。

| 指标 | 回答的问题 | 推荐做法 |
| --- | --- | --- |
| CLIP Score | 是否符合 prompt | 计算文本和生成图的 CLIP 相似度 |
| Style Similarity | 是否接近水彩训练集风格 | 计算生成图特征与训练集特征中心的相似度 |
| CLIP Diversity 或 LPIPS Diversity | 是否生成得太重复 | 计算生成图两两之间的平均距离 |
| Nearest Neighbor Similarity | 是否接近抄训练图 | 找每张生成图在训练集中的最近邻并可视化 |

### 8.1 指标解释方式

CLIP Score：

- 分数高说明 prompt 遵循能力较好。
- 微调后不应明显低于基础模型。

Style Similarity：

- 本项目最重要的指标。
- LoRA 模型应高于基础模型，说明水彩风格学习有效。

Diversity：

- 用于检查模型是否只会生成类似构图。
- 理想情况是 Style Similarity 上升，但 Diversity 不明显下降。

Nearest Neighbor：

- 用于检查是否过拟合或记忆训练图。
- 报告中建议展示 3 至 5 组“生成图 + 最近训练图”对比。

### 8.2 人工评分

人工评分作为补充即可，不需要太复杂。

建议让 5 至 10 人对每个模型随机抽取的生成图打 1 至 5 分，评分维度压缩为 3 个：

| 维度 | 说明 |
| --- | --- |
| 风格一致性 | 是否像水彩画 |
| Prompt 一致性 | 是否符合文字描述 |
| 视觉质量 | 是否清晰、自然、美观 |

## 9. 报告展示规划

报告中建议展示：

- 数据集样例图，说明水彩风格特征。
- 训练设置表格。
- 固定 prompts 和生成参数。
- Base 与不同 LoRA checkpoint 的同 prompt 对比图。
- 4 个核心指标表格。
- 最近邻可视化对比。
- 人工评分结果。
- 最终 checkpoint 选择理由。

## 10. 当前项目建议

建议把项目重点收窄为“水彩风格 LoRA 是否学得好”，不要把实验铺得太散。

具体建议：

- prompt 数量控制在 6 至 8 个，不需要覆盖太多类别。
- 数据收集优先保证风格一致性，其次才是主体多样性。
- 每个人的主训练集使用 `90 张公共图像 + 10 张个人图像`。
- 不建议只用 10 张个人图像做主模型训练。
- checkpoint 重点比较 500、1000、1500、2000 step。
- FID / KID 可以不做，核心指标放在 Style Similarity、CLIP Score、Diversity、Nearest Neighbor。
- 最终选择模型时，不选“风格最强”的 checkpoint，而选“风格提升明显、prompt 还能听、多样性没有明显崩”的 checkpoint。
